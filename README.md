# GeoAware-Core

GeoAware-Core is the canonical institutional memory and governance system for the GeoAware ecosystem.

This repository is documentation-first. It encodes the Constitution, decision records, AI collaboration model, product DNA, and operating doctrine that future humans and AI agents should inherit when working on GeoAware products.

## Operating Doctrine
Think in YCOMBINATOR Mode. Execute in GeoAware 80/20 Mode.

GeoAware uses YCOMBINATOR Mode to clarify the user, isolate the wedge, compress scope, and seek evidence. GeoAware uses 80/20 Mode to ship the smallest durable change that advances the mission without premature tooling or unnecessary complexity.

## What GeoAware-Core Protects
GeoAware-Core protects product identity, not implementation details. It should answer questions like:

- What makes a product unmistakably GeoAware?
- Which decisions define the ecosystem?
- What authority belongs to the Project Architect, AI collaborators, QA bots, and deployment systems?
- What kinds of features belong in GeoAware products?
- Which reviewer comments are blockers, and which should be bundled into later meaningful PRs?

## Core Principles
- GeoAware products should make users feel like explorers rather than consumers.
- Maps are not decoration. Maps are the product.
- Geography is the interface.
- AI supports exploration. AI never replaces the product experience.
- WaveAtlas is the canonical audio and radio engine for the GeoAware ecosystem.
- GeoAware Bible is a Geographic Bible Experience, not a generic Bible reader.
- Visible features must deepen exploration, geography, Scripture, journey, language, worship, or guidance.
- Non-blocking reviewer nits should be bundled into meaningful PRs unless they are blockers.
