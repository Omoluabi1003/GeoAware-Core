# PR Scorecard

Use this scorecard before merge.

## Required checks

- Product identity is protected.
- Geography remains central where relevant.
- Scope is small and coherent.
- No duplicate datasets or duplicate logic are introduced.
- No temporary architecture is added.
- Accessibility, performance, and security implications are considered.
- Non-blocking nits are bundled into future meaningful work.

## Merge recommendation

A PR should merge only when it improves the product without weakening the constitution.
